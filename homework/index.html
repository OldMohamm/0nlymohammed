1. If a station in CSMA/CD network is using the nonpersistent method, what can the station do if the medium is busy / idle? What is the advantage and disadvantage of the nonpersistent method?

CSMA/CD Nonpersistent Method

When medium is idle:

  • The station transmits immediately

When medium is busy:

  • The station waits for a random time period before sensing the channel again
  • It does not continuously monitor the channel during this waiting period
  • After the random wait, it senses the channel again and repeats the process

Advantages:

  • Reduces collision probability compared to 1-persistent method
  • Provides fairer access among competing stations
  • Better channel utilization under high load conditions
  • Minimizes the "clustering" of retransmissions

Disadvantages:

  • Increases delay compared to 1-persistent method
  • Potentially wastes channel capacity when no station transmits during random waiting periods
  • Additional complexity in implementation
  • Performance depends on choosing appropriate random backoff parameters

2. Explain why in OSI reference model the communication subnet boundaries include only the Physical, Data link and Network layers, not all the seven layers?

The OSI reference model includes only Physical, Data Link, and Network layers as subnet boundaries because:

  • Functional Separation: These three lower layers handle the physical transmission, framing, and routing of data which are the essential functions of a communication subnet
  • Network Infrastructure: They define the communication infrastructure independent of the upper layers, allowing for different network implementations to support the same higher-level protocols
  • Transparent Data Transfer: The subnet provides transparent data transfer, handling only delivery concerns without needing to understand the content or structure of the data
  • End-to-End Principle: Upper layers (Transport, Session, Presentation, Application) operate end-to-end between source and destination systems and are not concerned with the intermediate network elements
  • Performance and Efficiency: Implementing only necessary functions in network devices (like routers and switches) keeps them simpler and more efficient

This division also allows network designers to focus on connectivity issues at the lower layers while application developers can focus on higher-level functionality without worrying about the underlying network details.

3. In OSI reference model, layer x on one machine communicates with layer x on another machine as a peer-to-peer process. Explain why.

In the OSI reference model, layer x on one machine communicates with layer x on another machine as a peer-to-peer process for the following reasons:

  • Protocol Compatibility: Each layer implements protocols that communicate only with the same layer on another machine, ensuring compatible processing at both ends
  • Horizontal Communication Model: Information conceptually flows horizontally between peer layers on different machines
  • Virtual Communication: This creates virtual communication between peer layers, although no direct physical connection exists between them
  • Interface Standardization: The approach standardizes interfaces between layers, allowing independent implementation and evolution of each layer
  • Layered Abstraction: Each layer provides a specific set of services to the layer above and uses services from the layer below

While conceptually communication is horizontal between peer layers, actual data transfer occurs vertically through the protocol stack, then across the physical medium, and up through the protocol stack on the receiving machine.

4. In a CSMA/CD network with data rate 1Gbps, the maximum distance between any station pair is found to be 500m and the signal propagation speed is equal to 2/3 of the speed of light. To guarantee the detection of collision, find the maximum propagation time, minimum frame transmission time, and minimum frame size (in bytes).

CSMA/CD Maximum Propagation Parameters (1 Gbps)

Given:

  • Data rate = 1 Gbps = 109 bits/second
  • Maximum distance between stations = 500 m
  • Signal propagation speed = 2/3 × speed of light = 2/3 × 3×108 m/s = 2×108 m/s

1. Maximum propagation time:
The time for a signal to travel from one station to another at the maximum distance

Propagation time = Distance ÷ Propagation speed = 500 m ÷ (2×108 m/s) = 2.5 μs

2. Minimum frame transmission time:
For CSMA/CD to work properly, a station must be transmitting long enough to detect a collision

Minimum frame transmission time = 2 × Maximum propagation time = 2 × 2.5 μs = 5 μs

However, Ethernet standard requires a minimum frame transmission time of at least 512 bit times, which is:

512 bits ÷ 1 Gbps = 0.512 μs

Since 0.512 μs < 5 μs, the minimum frame transmission time must be 5 μs to guarantee collision detection.

3. Minimum frame size:

Minimum frame size in bits = Data rate × Minimum frame transmission time
= 109 bits/second × 5 μs = 5,000 bits

Minimum frame size in bytes = 5,000 bits ÷ 8 bits/byte = 625 bytes

Therefore:

  • Maximum propagation time = 2.5 μs
  • Minimum frame transmission time = 5 μs
  • Minimum frame size = 625 bytes

5. Consider a CSMA/CD network that transmits data at a rate of 10 Mbps over a 1km (kilometer) cable with no repeaters. If the minimum frame size in this network is 1250 bytes, what is the signal propagation speed (km/sec) of this cable?

Given:

  • Data rate = 10 Mbps = 10 × 106 bits/second
  • Cable length = 1 km = 1,000 m
  • Minimum frame size = 1,250 bytes = 1,250 × 8 = 10,000 bits

Step 1: Calculate the frame transmission time

Frame transmission time = Frame size ÷ Data rate
= 10,000 bits ÷ 10 × 106 bits/second = 0.001 seconds = 1 ms

Step 2: Apply the CSMA/CD constraint
For CSMA/CD to work, a station must be able to detect a collision before it finishes transmitting. This means:

2 × Propagation time ≤ Transmission time

Where propagation time is the time it takes for a signal to traverse the cable:

Propagation time = Cable length ÷ Signal propagation speed = 1,000 m ÷ v

Step 3: Solve for the minimum signal propagation speed (v)

2 × (1,000 m ÷ v) ≤ 0.001 seconds
2,000 m ÷ v ≤ 0.001 seconds
v ≥ 2,000 m ÷ 0.001 seconds
v ≥ 2,000,000 m/s = 2,000 km/s

Therefore, the signal propagation speed must be at least 2,000 km/s to ensure proper collision detection in this CSMA/CD network.

6. In Figure 1, the data rate is 10 Mbps, the distance between station A and Z is 2000 m, and the propagation speed is 2 × 108 m/s. Station A starts sending a long frame at time t1 =0, station Z starts sending a long frame at time t2 =3μs. The size of the frame is long enough to guarantee the detection of collision by both stations. Find:

  1. The time when station A hears the collision
  2. The time when station Z hears the collision
  3. The number of bits station A has sent before detecting the collision
  4. The number of bits station Z has sent before detecting the collision

Given:

  • Data rate = 10 Mbps = 10 × 106 bits/second
  • Distance between stations = 2,000 m
  • Propagation speed = 2 × 108 m/s
  • Station A starts transmission at t1 = 0
  • Station Z starts transmission at t2 = 3 μs

Step 1: Calculate one-way propagation time between stations

Propagation time = Distance ÷ Propagation speed
= 2,000 m ÷ (2 × 108 m/s) = 10 μs

i. Time when station A hears the collision:

The collision occurs when A's signal reaches Z at the same time Z is transmitting. Z started transmitting at t = 3 μs. Station A will detect the collision when Z's signal reaches A, which takes another 10 μs propagation time.

Time when A hears collision = t2 + Propagation time = 3 μs + 10 μs = 13 μs

ii. Time when station Z hears the collision:

The collision actually occurs at Z's location at time t = 10 μs (when A's signal reaches Z). Station Z will detect this collision immediately since it is already transmitting at its own location.

Time when Z hears collision = Propagation time = 10 μs

iii. Number of bits station A has sent before detecting the collision:

Bits sent by A = Data rate × Time before A detects collision
= 10 × 106 bits/s × 13 × 10-6 s = 130 bits

iv. Number of bits station Z has sent before detecting the collision:

Bits sent by Z = Data rate × (Time before Z detects collision - Z's start time)
= 10 × 106 bits/s × (10 - 3) × 10-6 s = 70 bits

  • The time when station A hears the collision: 13 μs
  • The time when station Z hears the collision: 10 μs
  • The number of bits station A has sent before detecting the collision: 130 bits
  • The number of bits station Z has sent before detecting the collision: 70 bits

7. In a network that implements the Ethernet protocol, what can the station do if the medium is busy/idle?

In an Ethernet network (which implements CSMA/CD), stations behave as follows:

When the medium is idle:

  • The station waits for the Interframe Gap (IFG) period (96 bit times or 9.6 μs in 10 Mbps Ethernet)
  • After the IFG, the station can begin transmission immediately

When the medium is busy:

  • The station must defer transmission until the channel is idle
  • Once the medium becomes idle, the station waits for the Interframe Gap (IFG)
  • After the IFG, if this is the first transmission attempt, the station transmits immediately
  • If this is a retransmission after a collision, the station applies the Binary Exponential Backoff algorithm:
    • Wait for a random number of slot times (where slot time = 512 bit times)
    • The random number is chosen from the range [0, 2k - 1], where k = min(n, 10) and n is the number of collision attempts
    • After 16 attempts, the transmission is aborted and an error is reported

During transmission, the station continues to monitor the channel for collisions. If a collision is detected:

  • The station transmits a 32-bit jam signal to ensure all stations detect the collision
  • The station increments its collision counter and schedules a retransmission using the backoff algorithm

8. Clarify the difference between the client-server and peer-to-peer models.

Aspect Client-Server Model Peer-to-Peer Model
Architecture Centralized with dedicated servers providing resources and services Decentralized where each node can be both client and server
Roles Clear separation between clients (request services) and servers (provide services) Each peer can both request and provide services simultaneously
Resource Sharing Resources are concentrated on and managed by servers Resources are distributed across the network with each peer contributing
Dependency Clients depend on servers; server failure affects all clients No central dependency; more resilient to single point failures
Management Centralized administration and control makes management easier Distributed management makes overall control more challenging
Scalability May require hardware upgrades of central servers to accommodate growth Scales naturally as new peers join and contribute resources
Security Better security through centralized control and enforcement More challenging to secure due to distributed nature and numerous access points
Performance Can become bottlenecked during high demand periods Performance can improve with more peers but may be unpredictable
Examples Web servers, email servers, database servers, file servers BitTorrent, early Napster, blockchain networks, distributed computing projects

9. What is the main difference between computer network and distributed system? Support you answer with example for each one.

Aspect Computer Network Distributed System
Definition A collection of autonomous computers connected by communication facilities A collection of independent computers appearing to users as a single coherent system
Primary Focus Connecting hardware and transfer of data between distinct devices Providing a unified computing environment that abstracts underlying complexities
User Awareness Users are aware of the multiplicity of machines and must explicitly access different systems Transparency is maintained - users are unaware of the underlying distribution of resources
System Management Each computer has its own operating system and is managed independently Systems are coordinated centrally with distributed components working together
Component Interaction Components interact through explicit communication protocols Components interact through a unified system interface and middleware
Example The Internet connecting various devices where users clearly access different websites on separate servers Cloud computing services like AWS where applications run on multiple servers but users interact with them as if they were a single system

Key Analogy:

A computer network is like a phone system where you have to dial different numbers to reach different people, while a distributed system is like calling a single company number where calls are routed internally without you needing to know the structure.
